# [https://nor.the-rn.info/rm_ation](https://nor.the-rn.info/rm_ation)

See `./CLAUDE.md` for more.

## Develop

The repo is around 1GB. Most of that is `/src/images/` (originals) and `/src/img-optimized/` (committed AVIF/WebP/JPEG derivatives generated by `@11ty/eleventy-img`). The derivatives are committed so Cloudflare deploys stay fast — see "Image workflow" below.

```zsh
git clone git@github.com:tyleretters/nor.the-rn.info.git
npm i
npm run dev
```

## Build

```zsh
npm run build
```

First build from a clean state takes ~6 minutes because `@11ty/eleventy-img` regenerates every derivative. Subsequent builds (and Cloudflare deploys with committed derivatives) take ~5 seconds.

## Image workflow

1. Add the original to `src/images/foo.jpg`
2. Reference it in markdown: `![alt](/images/foo.jpg)`
3. Run `npm run build` — eleventy-img generates derivatives in `src/img-optimized/`
4. Commit both the original AND the new files in `src/img-optimized/`

If you forget step 3 and push without derivatives, Cloudflare will regenerate them in its build container (slow but correct). The site won't break.

Remote images (CloudFront covers) and `/rm_ation/`-prefixed images are skipped by the transform — they pass through unchanged.

## Generating release posts

```zsh
npm run generate-release-posts
```

Scaffolds blog posts from new entries in `@tyleretters/discography`. See `scripts/generate-release-posts.js`.

## Search

[Pagefind](https://pagefind.app) builds a static search index during `npm run build` and powers the UI on `/search/`. No server required.

## Deploy

Automatic on push to `main` via Cloudflare Pages integration. Build settings configured in the Cloudflare dashboard.
